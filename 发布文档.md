# XUAD 发布与引用指南

本文档详细说明如何发布 XUAD 库到 JitPack 以及如何在其他 Android 项目中引用它。

## 1. 项目发布流程

本项目使用 [JitPack](https://jitpack.io) 进行构建和分发。JitPack 直接使用 GitHub 仓库作为发布源。

### 步骤一：提交代码与版本升级

1.  确保 `build.gradle` 中的 `version` 字段已更新（例如更新为 `4.0.0`）。
2.  提交所有代码更改到 Git 仓库。

### 步骤二：创建 Release / Tag

JitPack 依赖 Git 的 Tag 来确定版本。你必须在 GitHub 上创建一个 Release 或 Tag。

1.  打开项目的 GitHub 页面。
2.  进入 **Releases** 页面。
3.  点击 **Draft a new release**。
4.  在 **Choose a tag** 处输入版本号，例如 `4.0.0`（建议与 `build.gradle` 中的 version 保持一致）。
5.  点击 **Create new tag**。
6.  填写 Release title 和 Description（可选）。
7.  点击 **Publish release**。

### 步骤三：在 JitPack 构建

1.  访问 [https://jitpack.io/#xcwen98/xuad](https://jitpack.io/#xcwen98/xuad)。
2.  JitPack 会自动检测到新的 Release (`4.0.0`)。
3.  点击对应版本右侧的 **Get it** 按钮。
4.  等待 Log 图标变绿，表示构建成功。

---

## 2. 在其他项目中引用

构建成功后，你可以在任何 Android 项目中引用此库。

### 第一步：添加 JitPack 仓库

在项目的 **根目录** `settings.gradle` (推荐) 或 `build.gradle` 中添加 JitPack 仓库地址。

**settings.gradle (Kotlin DSL):**
```kotlin
dependencyResolutionManagement {
    repositoriesMode.set(RepositoriesMode.FAIL_ON_PROJECT_REPOS)
    repositories {
        google()
        mavenCentral()
        maven { url = uri("https://jitpack.io") } // 添加这一行
    }
}
```

**或者 build.gradle (Groovy):**
```groovy
allprojects {
    repositories {
        ...
        maven { url 'https://jitpack.io' }
    }
}
```

### 第二步：添加依赖

在具体的 **Module** (通常是 `app` 模块) 的 `build.gradle` 中添加依赖。

**build.gradle (Kotlin DSL):**
```kotlin
dependencies {
    implementation("com.github.xcwen98:xuad:4.0.0")
}
```

**build.gradle (Groovy):**
```groovy
dependencies {
    implementation 'com.github.xcwen98:xuad:4.0.0'
}
```

### 注意事项

*   **版本号**: `4.0.0` 对应你在 GitHub 上创建的 Tag 名称。
*   **快照版本**: 如果你想引用最新的提交（未打 Tag），可以使用 `master-SNAPSHOT` 作为版本号。
*   **Java 版本**: 本项目基于 JDK 17 构建，请确保你的项目也支持 JDK 17 (在 compileOptions 中设置 sourceCompatibility 和 targetCompatibility 为 JavaVersion.VERSION_17)。
